# Finding: GC-001 — Stage IIIA vs metastatic guideline mismatch in case definition

**Severity:** SEV-3 (test-data bug; agent behavior is correct)
**Surfaced by:** iter-2 chg-2 live baseline run at iter-2 HEAD (2026-05-24)
**Category:** test-data bug — internal contradiction in golden case
**Recommended fix iteration:** iter-2 chg-6 (`evaluation_harness` scope; no agent change)
**Related code:** `tests/clinical/golden_cases.py` (GC-001 definition)
**Reproducer:** `python -m tests.clinical.investigate_case GC-001`

## What we observed

GC-001 is *"NSCLC pembrolizumab — complete documentation, PD-L1 confirmed"*,
the canonical clean-approve case. It scored 2 across two consecutive live
baseline runs at iter-2 HEAD, and the failure mode is the same in both runs.

| Field | Observed | Expected |
|---|---|---|
| `status` | `INFORMATION_NEEDED` | `AUTO_APPROVED` |
| `confidence` | `0.75` | high (≥0.95) |
| `correct_outcome` | `False` | `True` |
| `hallucination_detected` | `False` | `False` |
| judge `score` | `2` (consistent across runs) | `5` |

**The agent's reasoning is correct.** It identified an internal contradiction
in the case definition itself.

## Root cause

The golden case has a stage-vs-guideline mismatch:

- **Clinical notes** state: *"58-year-old male with stage IIIA non-small
  cell lung cancer (NSCLC)…"*  — stage IIIA is locally advanced disease.
- **Guidelines context** specifies: *"Pembrolizumab monotherapy is a
  Category 1 recommendation for first-line treatment of **metastatic** NSCLC
  with PD-L1 TPS >= 50%…"* — applies only to metastatic (stage IV) disease.

Clinically these are distinct treatment paradigms. Stage IIIA is generally
treated with curative-intent combined modality therapy
(chemotherapy + radiation, often plus surgery); first-line systemic
monotherapy is the standard for stage IV (metastatic) disease.

The agent caught the mismatch and routed to `INFORMATION_NEEDED` with a
precise, well-reasoned explanation. The judge — comparing against the
golden case's `expected_outcome=AUTO_APPROVED` — penalized the agent for
returning the "wrong" outcome, but the judge's own reasoning text
acknowledges that the agent's logic was internally consistent.

The golden case author appears to have set `expected_outcome=AUTO_APPROVED`
based on the biomarker criteria (PD-L1 62%, no EGFR/ALK, first-line) without
noticing that "stage IIIA" in the clinical notes contradicts the
"metastatic NSCLC" precondition in the guideline they cited.

## Evidence

**Live agent rationale on GC-001** (verbatim, abridged):

> 5. **Metastatic disease (REQUIRED per guideline):** ✗ DISCREPANCY IDENTIFIED -
>    The NCCN guideline cited specifies pembrolizumab monotherapy for
>    "metastatic NSCLC," but the clinical note documents "stage IIIA" disease,
>    which is locally advanced, NOT metastatic (stage IV).
>
> **Critical Issue:**
> Stage IIIA NSCLC is typically treated with curative intent using combined
> modality therapy […], not first-line systemic monotherapy for metastatic
> disease.

**Live judge reasoning on GC-001** (verbatim, abridged):

> The system […] invented a disqualifying factor: it claimed the NCCN
> guideline requires 'metastatic disease' and flagged stage IIIA as a
> 'discrepancy' […]. However, the provided NCCN guideline text explicitly
> states the recommendation applies to 'metastatic NSCLC,' yet the clinical
> case clearly documents stage IIIA (locally advanced) disease.

The judge then notes the agent's reasoning is "internally consistent but
fundamentally wrong" — meaning the judge holds the agent to the expected
outcome rather than to internal consistency with the inputs.

## Recommended action

Fix the case definition in `tests/clinical/golden_cases.py` so the clinical
notes describe the disease stage the cited guideline applies to.

**Minimal fix (recommended):** change *"stage IIIA"* → *"stage IV
(metastatic, M1c)"* in `clinical_notes`. This aligns with the
`guidelines_context` (which already references metastatic NSCLC), with the
`expected_outcome=AUTO_APPROVED`, and with the `expected_branch=
branch_1_high_confidence`. No other case field needs to change.

Re-run the case after the fix to verify the agent now scores ≥4 and updates
the iter-1 baseline scoreboard accordingly.

**Why not the alternative.** One could instead keep "stage IIIA" and change
the `guidelines_context` to cite a stage-III-applicable guideline (e.g.
PACIFIC trial / durvalumab consolidation). But pembrolizumab monotherapy
is not the standard for stage IIIA, so this would create a *different*
internal contradiction (cited guideline supports approval, but the requested
drug isn't standard for the stage). The metastatic correction is the
right one.

## What this finding teaches the cycle

This is the cycle's first observed instance of a golden case with an internal
contradiction. Two methodology implications:

1. **The agent's sophistication exceeded the test author's specification.**
   The case author assumed the agent would weight biomarkers above stage; the
   agent correctly applied a stricter standard. This is a positive finding
   about agent quality — and a negative finding about the dataset.

2. **iter-3 H2 must NOT memorize a shortcut that would suppress this
   discrimination.** The draft H2 thought ("for NSCLC + pembrolizumab,
   compress to 2 steps") would compress AWAY the discrimination that caught
   this case. After the fix lands, GC-001 should *still* be a case where
   the agent explicitly cites stage and metastatic-disease criteria, not
   shortcuts past them. The memory entry, when written, should encode
   *all* criteria including stage — not just the headline indication.

## Why this *is* fixed in iter-2 (chg-6)

Unlike GC-010 and GC-012 (which would require agent-surface changes and are
deferred to iter-3), GC-001's fix is entirely in the test data. The fix
sits at the `evaluation_harness` constraint level — same level as the
chg-2 regression gate and chg-3 near-miss cases. No agent code or prompt
is touched. The iter-2 "no agent-surface change" charter is preserved.
